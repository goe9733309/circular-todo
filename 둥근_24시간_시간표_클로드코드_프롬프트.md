# 둥근 24시간 시간표 — Claude Code 3단계 구현 프롬프트

PRD(`둥근_24시간_시간표_PRD.md`) 기준. 아래 3개 프롬프트를 순서대로 Claude Code에 붙여넣어 진행합니다.
결과물은 단일 HTML 파일(`round_clock.html`) 하나로 계속 이어서 수정하는 방식입니다.

---

## Phase 1 프롬프트 — 핵심 시계 + 할 일 목록 (뼈대)

```
너는 "둥근 24시간 시간표"라는 개인용 도구를 만들 거야. 단일 HTML 파일 하나(round_clock.html)로
완결되는 결과물이고, 인라인 <style>/<script>만 사용해. 외부 서버나 백엔드는 없고, 브라우저에서
파일을 열면 바로 동작해야 해.

## 레이아웃
- 화면을 좌우 2분할: 좌측(60~65%)은 둥근 24시간 시계판, 우측(35~40%)은 시간 설정 폼 + 할 일 목록.
- 데스크톱은 좌우 배치, 화면 폭이 좁아지면(모바일) 시계가 위, 목록이 아래로 오는 세로 배치로 전환
  (media query 기준 폭 768px 정도로 잡아줘).

## 좌측: 둥근 24시간 시계판 (SVG)
- SVG로 원형 시계판을 그린다. 0시가 정각 맨 위(12시 방향)에 오고, 시계 방향으로 0~24시가 한 바퀴를
  돈다 (즉 1시간 = 15도).
- 1시간 단위 큰 눈금 24개, 각 눈금 옆에 숫자 라벨(0, 1, 2, ... 23) 표기.
- 10분 단위 보조 눈금(작은 선)도 추가.
- 시계판 중앙에 원 하나(문자반), 그 위에 "긴 바늘" 하나를 배치. 바늘은 현재 시각(시+분+초를 모두
  반영한 정확한 각도)을 가리키고, 1초마다 갱신되어 부드럽게 움직인다. CSS transform: rotate()를
  중심점 기준으로 사용해서 매끄럽게 처리해줘.
- 시계판 정중앙 근처(바늘에 가리지 않는 위치, 예: 원 아래쪽 여백)에 현재 디지털 시각(HH:MM:SS)을
  텍스트로 표시.
- 나중에(Phase 2~3에서) 시간대 밴드와 알람 마커를 추가할 것이므로, 이를 위한 SVG 그룹(<g id="event-bands">,
  <g id="alarm-markers"> 등)을 미리 비워둔 채로 구조만 잡아둬.

## 우측: 시간 설정 폼
- 시작 시각, 종료 시각 입력 (HTML <input type="time">로 24시간 표기).
- 할 일 이름 입력 (텍스트).
- 색상 선택 (color input 또는 미리 정의된 6~8개 색상 스와치 중 클릭 선택 — 파스텔 톤으로).
- "추가" 버튼.
- 유효성 검사: 시작 < 종료여야 하고, 아니면 안내 메시지.

## 우측: 할 일 목록
- 등록된 항목을 시작 시각 기준 오름차순으로 리스트 표시.
- 각 항목: "HH:MM - HH:MM  할일명" + 삭제 버튼(수정은 Phase 2에서).
- 항목을 추가/삭제하면 즉시 목록 갱신.
- 시각 표시 형식은 24시간제로 통일 (예: 09:00, 14:30).

## 데이터 & 저장
- 이벤트 데이터는 PRD 6장 데이터 모델을 참고해서 JS 배열로 관리하고, localStorage에 JSON으로 저장/복원.
  키는 "roundClock_events_YYYY-MM-DD" 형태로 날짜별로 구분해줘 (자정 넘어가면 자연스럽게 새 목록이
  되도록 하기 위함, 단 자정 리셋 로직 자체는 Phase 3에서 다듬을 예정이니 지금은 날짜별 키 구조만
  잡아두면 됨).
- 새로고침해도 오늘 등록한 항목이 유지되어야 함.

## 스타일 (지금 단계에서는 기본만)
- 지금 단계에서는 손그림 스타일을 완성하지 않아도 됨(Phase 3에서 다듬음). 다만 전체적으로 깔끔하고
  읽기 쉬운 기본 스타일(적당한 여백, 둥근 모서리, 파스텔 배경 정도)로 만들어줘.
- 폰트는 시스템 기본 sans-serif로 우선 두고, Phase 3에서 손글씨 폰트로 교체할 예정.

## 완료 기준
- 파일을 브라우저로 열면 좌측에 24시간 원형 시계가 보이고, 바늘이 현재 시각을 정확히 가리키며
  실시간으로 움직인다.
- 우측 폼으로 시간대+할일을 등록하면 목록에 즉시 나타나고, 새로고침해도 유지된다.
- 잘못된 시간 범위 입력 시 에러 안내가 뜬다.
```

## Phase 2 프롬프트 — 시간대 밴드 시각화 + 알람 기능

```
Phase 1에서 만든 round_clock.html을 이어서 작업해. 이번 단계는 시계판에 할 일 시간대를 시각적으로
표시하고, 알람(브라우저 알림) 기능을 추가하는 거야.

## 1) 시계판에 시간대 밴드(arc) 표시
- 등록된 각 할 일 항목의 시작~종료 시각 구간을, 시계판 눈금 바로 안쪽(또는 바깥쪽)에 색칠된 호
  (arc band) 형태로 그려줘. 색상은 폼에서 선택한 색을 그대로 사용.
- SVG path의 arc 명령(A)을 이용해서 시작 각도~종료 각도 사이의 링 모양 밴드를 그리는 함수를 작성해줘
  (각도 계산: 0시=0도 기준, 1시간=15도, 분 단위도 비례 반영).
- 같은 시간대에 항목이 여러 개 겹치면(안 겹치는 게 정상이지만 방어적으로) 밴드가 겹쳐 그려져도
  괜찮음(우선순위 없음, 나중 항목이 위에 그려지는 정도로 충분).
- 밴드에 마우스를 올리면(hover) 해당 할 일의 이름과 시간 범위를 툴팁으로 보여줘 (간단한
  title 속성 또는 커스텀 툴팁 div 둘 다 가능, 구현하기 쉬운 방식으로).
- 항목을 추가/삭제할 때마다 밴드도 즉시 다시 그려져야 함.

## 2) 알람 설정
- 시간 설정 폼에 "알람 설정" 체크박스를 추가. 체크되어 있으면 이 항목의 시작 시각에 알림이 울리도록
  등록.
- (선택) 체크박스 옆에 라디오나 드롭다운으로 "시작 시각에 울림 / 종료 시각에 울림" 선택 가능하게
  해줘 (PRD의 alarmTriggerPoint 필드 활용).
- 할 일 목록의 각 항목 옆에 알람 설정 여부를 아이콘이나 텍스트(🔔 표시 등)로 표시.

## 3) 알람 트리거 로직
- 최초 페이지 로드 시 브라우저 Notification API 권한을 요청 (Notification.requestPermission()).
- 10~30초 주기로 현재 시각을 체크해서, 알람이 설정된 항목의 트리거 시각(시작 또는 종료)과 현재
  시각의 시:분이 일치하면 알림을 발생시켜.
  - 권한이 허용된 경우: new Notification()으로 브라우저 알림 표시 (제목: 할 일 이름, 본문:
    시간 범위).
  - 권한이 거부되었거나 지원 안 되는 경우: 화면 내 토스트/모달 팝업으로 동일한 내용을 보여주는
    fallback을 반드시 구현해줘.
- 같은 알람이 하루에 여러 번 울리지 않도록, 트리거된 항목은 alarmFiredToday 같은 플래그를
  true로 바꿔서 중복 발생을 막아줘 (localStorage에도 반영).
- 간단한 알림음(beep 등)을 Web Audio API나 짧은 base64 오디오로 재생 (외부 파일 의존 없이).

## 4) 시계판 위 알람 마커
- 알람이 설정된 항목의 트리거 시각 위치에, 눈금 바깥쪽으로 작은 마커(별표 모양 정도로 충분,
  텍스트 이모지 "⭐"나 간단 SVG path 사용)를 표시해줘. 정교한 손그림 아이콘은 Phase 3에서 다듬을
  예정이니 지금은 위치와 동작이 정확한 것에 집중.

## 완료 기준
- 등록한 할 일 시간대가 시계판에 색칠된 호로 나타나고, hover하면 내용이 보인다.
- 알람 체크박스를 켠 항목의 트리거 시각이 되면 브라우저 알림(또는 fallback 팝업)과 소리가 울린다.
- 같은 알람이 하루 중 중복으로 울리지 않는다.
- 새로고침해도 알람 설정/발생 여부가 유지된다.
```

## Phase 3 프롬프트 — 손그림 스타일 마감 + 반응형/자정 처리 디테일

```
Phase 2까지 완료된 round_clock.html을 이어서, 이번 단계는 "손으로 그린 듯한" 비주얼 완성과
디테일 다듬기에 집중해줘. 기능 로직은 거의 그대로 두고 스타일/마감 위주로 작업해줘.

## 1) 손그림 스타일 비주얼
- Google Fonts에서 한글/영문 손글씨 느낌 폰트를 CDN으로 불러와 전체 UI(숫자 라벨, 할 일 텍스트,
  버튼, 목록)에 적용해줘 (예: 'Gaeguli', 'Nanum Pen Script', 'Single Day' 등 사용 가능한
  손글씨 계열 중 골라서 — 한글이 포함되므로 한글 지원 폰트를 우선해줘).
- 시계판 원, 눈금선, 바늘을 완벽한 직선/원이 아니라 살짝 삐뚤빼뚤한 손그림 느낌으로 만들어줘.
  방법 예시: SVG filter의 feTurbulence + feDisplacementMap을 원/선에 적용하거나, path의 좌표를
  약간씩 랜덤하게 흔들어 손그림 느낌의 stroke를 만드는 방식 중 구현하기 쉬운 쪽으로 선택해줘.
- 배경은 종이/노트 질감 느낌의 베이지색 계열로.
- 버튼, 입력창 테두리도 각진 사각형 대신 살짝 삐뚤어진 손그림 테두리 스타일로 (border-radius를
  불규칙하게 주거나, 배경에 손그림 느낌 테두리 SVG를 겹치는 방식 등).
- 색상 팔레트는 색연필/크레파스 느낌의 채도 낮은 파스텔 톤으로 전체 통일.
- 알람 마커(별표)도 손그림 스타일의 간단한 SVG 아이콘으로 교체해줘.

## 2) 반응형 마감
- 768px 이하에서 좌우 배치를 상하 배치로 전환하는 부분을 다시 한번 점검하고, 시계판 크기가
  화면 폭에 맞춰 적절히 줄어들도록(viewBox 활용) 조정해줘.
- 모바일에서 터치로 hover 툴팁이 잘 안 보이는 문제가 있을 수 있으니, 터치 시 탭하면 툴팁이
  나타났다가 일정 시간 후 사라지는 방식으로 보완해줘.

## 3) 완료/자정 처리
- 목록에서 이미 종료 시각이 지난 항목은 흐리게(opacity 낮춤) 표시하거나 취소선을 적용해줘.
- 현재 진행 중인 항목(현재 시각이 시작~종료 사이)은 목록에서 배경색 강조로 표시해줘.
- 자정이 지나면(날짜가 바뀌면) 새로운 날짜의 빈 목록으로 자연스럽게 전환되고, 알람 발생 플래그도
  새 날짜 기준으로 초기화되도록 로직을 점검/보완해줘 (Phase 1에서 잡아둔 날짜별 localStorage 키
  구조를 활용).

## 4) 기타 마감
- 첫 방문 시 알림 권한을 아직 허용/거부하지 않은 상태라면, 화면 한쪽에 손그림 스타일의 작은
  안내 배너("🔔 알람을 받으려면 알림을 허용해주세요")를 보여주고, 허용/거부 후에는 배너가
  사라지도록 해줘.
- 전체적으로 콘솔 에러 없이 깔끔하게 동작하는지 마지막으로 점검해줘.

## 완료 기준
- 전체 UI가 손으로 그린 듯한 일관된 스타일(폰트, 선, 색상, 아이콘)로 통일되어 있다.
- 모바일/데스크톱 모두에서 레이아웃이 깨지지 않는다.
- 지난 항목/진행 중 항목이 목록에서 시각적으로 구분된다.
- 자정이 지나면 알람/완료 상태가 새 날짜 기준으로 정상 초기화된다.
```
