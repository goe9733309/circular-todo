# 둥근 24시간 시간표 (Hand-drawn Circular Clock) PRD

## 1. 개요

### 1.1 배경
하루 24시간을 하나의 원판 위에 표현하는 손그림(hand-drawn) 스타일 시간표. 왼쪽에는 24시간 원형 시계판, 오른쪽에는 시간대별 할 일 설정과 목록을 배치한다. 긴 바늘이 현재 시각을 실시간으로 가리키며, 사용자가 지정한 목표 시각에 도달하면 알람으로 알려준다.

### 1.2 목적
- 하루 전체를 한눈에 조망하는 원형 타임라인 제공
- 종이에 손으로 그린 듯한 따뜻하고 아날로그적인 느낌의 UI
- 단순 시계가 아니라 "시간대별 할 일 + 알람"이 결합된 개인 스케줄 도구

### 1.3 형식
기존 작업 패턴과 동일하게 **단일 HTML 파일** (인라인 CSS/JS, 외부 서버 불필요, 브라우저에서 바로 실행).

## 3. 핵심 기능 (Functional Requirements)

### 3.1 둥근 24시간 시계판 (좌측)
- FR-1: 원판에 0~24시를 시계 방향으로 표시 (0시/24시가 정각 위쪽, 12시가 정각 아래쪽 — 또는 사용자가 원하는 기준점 선택 가능. 기본은 0시=위쪽).
- FR-2: 1시간 단위 눈금 24개 + 5분 또는 10분 단위 보조 눈금.
- FR-3: 각 시간 눈금에 숫자 라벨(0, 1, 2, ... 23) 표기, 손글씨 폰트 사용.
- FR-4: **긴 바늘 하나**가 현재 시각을 실시간으로 가리킴 (분 단위까지 반영해 부드럽게 이동, 1초/1분마다 갱신).
- FR-5: 바늘은 중심에서 원 가장자리 근처까지 뻗는 형태, 손그림 느낌의 약간 삐뚤빼뚤한 선(스케치 필터) 적용.
- FR-6: 원판 위에 등록된 "할 일 시간대"를 호(arc) 형태의 색칠된 밴드로 표시 (예: 09:00~10:00 구간에 색칠된 띠).
- FR-7: 목표 알람 시각이 있으면 해당 지점에 작은 마커(별표, 깃발 등 손그림 아이콘) 표시.
- FR-8: 시계판 중앙에 현재 디지털 시각(HH:MM:SS) 텍스트 표시.
- FR-9: 시간대 밴드나 마커에 마우스를 올리면(hover) 할 일 내용 툴팁 표시.

### 3.2 시간 설정 (우측 상단)
- FR-10: 시작 시각, 종료 시각을 입력하는 폼 (드롭다운 또는 time input, 24시간 표기).
- FR-11: 할 일 이름/메모 입력란.
- FR-12: 색상 선택(원판에 표시될 밴드 색) — 손그림 색연필 팔레트 느낌의 제한된 색상 중 선택.
- FR-13: "알람 설정" 체크박스 — 체크 시 해당 시간대 시작 또는 종료 시점에 알림 발생.
- FR-14: 입력 후 "추가" 버튼으로 목록과 원판에 즉시 반영.

### 3.3 할 일 목록 (우측 하단)
- FR-15: 등록된 항목을 시간 순으로 정렬해 리스트로 표시 (시작-종료 시각, 할 일명, 알람 여부 아이콘).
- FR-16: 각 항목에 수정/삭제 버튼.
- FR-17: 현재 진행 중인 항목은 리스트에서 강조 표시(하이라이트).
- FR-18: 지나간(완료된) 항목은 흐리게 표시하거나 취소선.

### 3.4 알람/알림 기능
- FR-19: 등록된 목표 시각(또는 시간대 시작 시각)에 도달하면 브라우저 알림(Notification API) 또는 화면 내 팝업 + 소리로 알려준다.
- FR-20: 최초 사용 시 브라우저 알림 권한 요청.
- FR-21: 알림 미허용 시에도 화면 내 모달/토스트 알림은 항상 동작.
- FR-22: 같은 알람이 중복으로 울리지 않도록 트리거 후 상태 기록(하루 1회).
- FR-23: 브라우저 탭이 백그라운드에 있어도 setInterval 기반으로 지속 체크 (탭이 완전히 닫히면 알람은 동작하지 않음 — 한계점으로 명시).

### 3.5 데이터 저장
- FR-24: 등록한 시간대/할 일/알람 설정은 localStorage에 저장해 새로고침 후에도 유지.
- FR-25: 날짜가 바뀌면(자정 경과) 완료 표시나 알람 트리거 상태를 초기화.

## 4. 비주얼 스타일 (손그림 느낌)

- 전체적으로 종이 질감 배경(베이지/크림색) 또는 노트 느낌.
- 선은 완벽한 직선/원이 아니라 손으로 그린 듯 약간 흔들리는 스타일 — SVG `filter` (turbulence/displacement) 또는 수작업으로 좌표를 미세하게 흔든 path 사용.
- 폰트: 손글씨 스타일 웹폰트 (예: Google Fonts의 `Gaeguli`, `Nanum Pen Script`, `Single Day` 등 한글/영문 손글씨체) 적용.
- 색상: 색연필/크레파스 느낌의 채도 낮은 파스텔 팔레트.
- 아이콘(알람 종, 별, 깃발 등)도 손그림 스타일 SVG로 직접 제작.
- 버튼/입력창도 각진 사각형이 아니라 살짝 삐뚤어진 손그림 테두리 스타일.

## 5. 기술 스택 및 구조

- **단일 HTML 파일** (index.html), 인라인 `<style>`/`<script>`.
- 시계판: SVG 기반 렌더링 (원, 눈금, 바늘, 호(arc) 밴드 모두 SVG path/circle로 구현 — 손그림 필터 적용이 용이).
- 로직: Vanilla JavaScript (프레임워크 불필요).
- 저장: `localStorage` (JSON 직렬화).
- 알림: 브라우저 `Notification` API + fallback 인앱 토스트.
- 시간 갱신: `setInterval` (1초 단위 바늘 갱신, 알람 체크는 별도 주기 예: 10~30초).
- 외부 폰트: Google Fonts CDN 링크.
- 별도 서버/백엔드 없음 (기존 API 호출 패턴과 달리 이 도구는 외부 API 불필요).

## 7. 비기능 요구사항

- 반응형: 데스크톱(좌우 배치) / 모바일(상하 배치) 모두 대응.
- 성능: 바늘 애니메이션은 CPU 부담 없이 부드럽게 (transform 기반 회전 권장).
- 접근성: 색상 외에 텍스트 라벨 병기(색약 사용자 고려).
- 브라우저 알림 미지원/미허용 환경 대비 fallback 필수.
- 오프라인에서도 전체 기능 동작 (외부 API 의존 없음).

---

## 8. 제약 및 한계 (명시적으로 알려야 할 부분)

- 브라우저 탭/창이 완전히 닫히면 알람이 울리지 않음 (Notification API의 한계 — 백그라운드 서비스가 아님).
- localStorage는 브라우저/기기별로 분리되어 저장됨 (기기 간 동기화 없음).
- 24시간을 한 바퀴로 표현하므로, 12시간 아날로그 시계보다 눈금 간격이 촘촘함 — 가독성 확보를 위한 폰트 크기/여백 설계 필요.

## 9. 열린 질문 (진행 전 확인 필요)

1. 0시(자정) 위치를 원판 맨 위로 고정할지, 다른 기준(예: 현재 시각이 항상 위로 오는 회전판 방식)을 원하는지?
2. 알람 사운드는 기본 제공 효과음을 쓸지, 파일 업로드를 지원할지?
3. 할 일 항목 개수 제한이 있는지 (하루 몇 개까지 예상하는지)?
4. 여러 날짜를 넘나들며 볼 수 있어야 하는지, 아니면 "오늘"만 다루면 되는지?
5. 모바일 홈 화면에 추가해 쓰는 PWA 형태까지 고려할지, 아니면 단순 HTML 파일 실행으로 충분한지?

## 10. 단계별 진행 제안 (Milestones)

- **Phase 1 (핵심 시계 + 목록)**: SVG 원형 24시간 시계판, 실시간 바늘, 시간대 입력 폼, 할 일 목록 CRUD, localStorage 저장. 손그림 스타일 기본 적용.
- **Phase 2 (알람 기능)**: Notification API 연동, 인앱 알림 fallback, 알람 트리거 로직 및 중복 방지, 사운드 재생.
- **Phase 3 (디테일 다듬기)**: 손그림 스타일 정교화(SVG 필터, 손글씨 폰트, 아이콘 제작), 반응형 레이아웃, 자정 리셋 로직, 완료 항목 표시.
